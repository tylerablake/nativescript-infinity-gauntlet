Space

  • Framework to build truly native cross platform mobile apps
  • Open Source, backed by Progress
  • Multiple flavors (Vanilla, Angular, and Vue)
  • Wait... but truly native apps?
    • Access to native objc/Java classes
    • Provides native performance
    • Enables the same customization as native

It has multiple flavors

  • Vanilla - Plain JS and/or TypeScript
  • Angular
  • Vue
  • React is in the works as well

Truly native apps? Yes!

You have access to all native API's (objc and java)

This results in native performance

As well as providing the same amount of customization as native

Reality

You write JS/TS code

It runs inside of the device's JavaScript VM

  • Android - V8
  • iOS - JavaScriptCore

The NS runtime generates metadata which tells what native API's are available

The JS code is ran through a type conversion process to convert it to native API's

The converted code is sent to the call dispatcher to call the native API's

Time

NativeScript doesn't use HTML or the DOM, it uses it's own XML-esque markup

In NativeScript everything must be inside of a container


There are 6 layout containers

  • Absolute (define the x,y position for the top left of the layout)
  • Dock (force the elements to dock to the top, bottom, left, or right of the layout)
  • Grid (similar to Bootstrap but you get to define your column and row sizes, etc)
  • Stack (stacks elements on top of eachother or beside each other)
  • Wrap (similar to StackLayout but wraps instead of stacking)
  • Flexbox (a non-conforming implementation of the CSS Flexible Box Layout)

Now ScrollView is NOT a layout container but can be treated like one
